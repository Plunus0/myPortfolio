[개념적 데이터모델링]
ER다이어그램
현실세계 개체들의 관계를 표현한다.
무결성의 원칙을 지킬 수 있다.
개체, 속성, 관계, 관계의 속성

Entitiy : 실체가 있는 개체를 테이블화 한 것(ㅁ으로 그려짐)
properties : 속성, Entity를 나타내는 속성들을 나타낸다. 단순(단일)/복합(다중) 속성으로 나뉘며, 유일한 속성은 식별자(Primary key)로 설정 후 ER다이어그램에선 밑줄표기 (ㅇ로 그려짐)

ER다이어그램 작성시 관계의 종류에 따른 표현방식(erd.yah.ac)
관계는 마름모(<>)로 표현

Cardinality(관계유형 : 관계의 정의)
1:1 관계 : - (보통 종속적인 키가 자식이며, 외래키로 지정)
1:N 관계 : -<, >- (보통 N관계가 외래키로 지정됨)
N:M 관계 : >-< (각 테이블의 정보를 바탕으로 별도의 맵핑 테이블을 작성 두 테이블에서 끌어온 정보를 pk 및 fk 로 지정 가능)
유도속성 : 점선(정보를 토대로 유추할 수 있는 것)
1:1 비식별관계 : 점선
1:N 비식별관계 : 점선
다중속성 : 2중 실선

Mandatory : 반드시 필요로 하는 조건(min 1) (필수적인 곳에 ㅁ+-)
Optionality : 조건이 필수적이지 않은 것(min 0) (필수적이지 않은 곳에 -ㅇㅁ로 이음)

[논리적 모델링] - ER Master
1) 사상규칙과정 : 개념적 모델링을 한 것을 ER다이어그램을 한것을 관계형 모델을 스키마로 변환하는 과정
기본키(primary key) : 한 테이블내에서 튜플간 중복되지 않는 값을 저장하고 있는 속성의 집합
* 약한 개체에 기본키를 설정할 경우 강한 개체를 같이 묶어 지정한다.
외래키(foreign key) : 튜플간 관계를 표현하기 위해, 한 테이블의 기본키를 다른 테이블에서 참조하고 있는 속성(의존하게 되는 키를 설정)

Entity(ㅁ) -> Table
속성 Attribute (ㅇ) -> 컬럼(열)
개체내부 데이터->튜플(행)
관계 Relation(<>) -> PK, FK

2) 정규화과정(NORMALIZATION) : 개체들의 이상현상을 제거해서 DBMS가 효율적으로 데이터를 처리하도록 관계스키마를 형성하는 것 (쓰기를 위해 읽기의 성능을 희생) (제 3 정규화 초과 부터는 학술적으로 이용된다.)

제 1 정규화 : 각 속성값을 원자값으로 만든다. (더이상 나눌 수 없다.)
제 2 정규화 : 기본키에 속하지 않는 속성 모두가 기본키에 완전함수 종속적이어야 한다.(완전함수 종속성을 갖도록 부분 함수적 종속성을 제거해야 한다.)
제 3 정규화 : 제 2 정규화 이면서 기본키에 속하지 않는 모든 속성이 기본키에 이행적 종속적이지 않은 릴레이션을 의미한다.

[물리적 데이터 모델링]
역정규화(denormalization) : 정규화를 통해 만든 표를 성능이나 개발의 편의성을 위해 조작하는 것 (읽기를 위해 쓰기의 성능을 희생), 여러대의 서버를 이용할 때 사용
* 방법1(컬럼의 역정규화)
컬럼중복(join을 줄이기) : 중복된 값을 허용하더라도 join을 줄여 한번에 출력되게 만든다.
파생컬럼의 형성(계산작업을 줄이기) : 출력시마다 계산해야할 작업이 있다면 미리 column 및 계산결과를 저장해둔다.
컬럼을 기준으로 테이블을 분리 :  비효율적이나 최후의 수단으로 쓰이며 여러 컴퓨터에 분산저장할 수 있도록 분리한다.
행을 기준으로 테이블 분리 : 행에 저장된 값의 종류가 많을 경우 이를 쪼개서 각기 다른 테이블에 저장한다.
* 방법2(관계의 역정규화)
지름길을 연결 : 관련있는(자주 사용되는) 테이블간의 조인을 외래키를 이용하여 조인없이 정보를 연결할 수 있도록 한다.
