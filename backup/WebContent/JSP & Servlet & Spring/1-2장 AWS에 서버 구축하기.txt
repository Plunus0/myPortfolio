<AWS>
Amazon이 제공하는 cloud service, 관리가 쉽고 빠르고 유연한 확장성 및 보안이 장점
필요할 때만 사용하고, 사용한 만큼만 비용 지불
* 최초 가입시 12개월 동안 무료사용 가능
* 사용 후 중지 하지 않으면 사용이 지속된다.
* 작업 > 인스턴스상태 > 중지
* 종료를 누르면 인스턴스가 삭제된다.(주의)
* 사용시간 'Billing' 검색

- AWS의 종류
> EC2 : 크기 조정이 가능한 컴퓨팅 용량을 클라우드에서 제공하는 웹 서비스
> S3 : 확장성, 가용성, 내구성을 가진 데이터 저장공간(Simple Storage)를 제공
> RDS : 관계형 DB관리 서비스, 관계형 DB(MySQL, ORacle 등)를 모니터링, 주기적 백업

- 관련용어
on-Premise : 관리자가 물리적인 서버를 직접 운영하는 방식(IDC 이용 / AWS-EC2 이용 등이 해당)
Serverless : 서버 작업을 서버내부가 아닌 클라우드 서비스로 처리(AWS-RDS 등이 해당)
Region : 데이터센터가 물리적으로 존재하는 곳
CDN(Contect Delivery Network) : 정적 리소스를 빠르게 제공할 수 있게 전 세계의 캐시서버에 복제해주는 서비스

- AWS에 서버 구축하기 : 서비스 선택
1. 메인화면에서 모든서비스 > EC2 선택
2. 우측 상단에서 Region변경(서울) 후 인스턴스 시작

- AWS에 서버 구축하기 : 인스턴스 생성
1. AMI 선택 : Windows(프리티어 사용가능)
2. 인스턴스 유형 선택 : 검토 및 시작
3. 인스턴스 시작 검토 : 시작하기
4. 새 키 페어 생성 : 키 페어 이름 입력(다운로드 받아놓기)
5. 시작 상태 : 인스턴스 보기

- AWS에 서버 구축하기 : RDP 실행
1. 실행할 인스턴스 선택 후 연결
2. RDP 클라이언트 tab > 원격 데스크톱 파일 다운로드
*MAC 사용자의 경우 appStore에서 rdp 검색 및 다운로드
3. 원격 데스크톱 파일 실행 > 사용자이름 / 암호 입력
* 키 페어 이름 설정 시 다운로드 받았던 파일을 암호 가져오기 > 키 페어로 이동에 업로드 > 암호 해독
4. Spring 개발도구 설치 및 설정
4-1. 윈도우 방화벽 설정 : 윈도우 방화벽 > 고급 설정 > 인바운드 규칙 > 새 규칙 > 포트 > 8080 > 이름 : tomcat tcp8080
4-2. AWS 방화벽 설정 : EC2대시보드 > 보안그룹 > launch-wizard-1 클릭 > 인바운드 규칙 > 인바운드 규칙 편집 > 유형 : 모든 TCP
* 인스턴스를 종료 후 재시작할 경우 IP주소가 변경되므로 원격 데스크톱 파일을 다시 다운받아야함

- AWS에 서버 구축하기 : 외부에서 AWS내의 tomcat에 접속하기
EC2대시보드 > 인스턴스 > 네트워킹 > 퍼블릿 IPv4 주소 복사 > 브라우저주소창 : 퍼블릿 IPv4주소:포트번호(8080) 입력
